# Version Control Task

## Version Control
    Version control is a system that helps developers track changes made to code over time. It allows you to go back to previous versions, collaborate with others, and avoid losing work.

---

## Difference Between Git and GitHub
    1. Git is a version control system while, GitHub is a platform for hosting Git repositories.

    2. Git works locally on your computer while, Github works online (cloud-based).
    
    3. Git is used to track changes in code while, Github is used to collaborate and share code 

---

    3. SourceForge

## Difference Between git fetch and git pull

### git fetch: Gets updates from the remote repository but does NOT merge them into your current branch.

### git pull: Gets updates AND automatically merges them into your current branch.

---

## Git Rebase
    Git rebase is used to move or combine a sequence of commits to a new base commit. It helps keep your project history clean and linear.

### Command:
    git rebase branch-name.

## cherry-pick commit-hash
Git cherry-pick allows you to pick a specific commit from one branch and apply it to another branch.

### Command:
git cherry-pick commit-hash
